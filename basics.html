What is Git vs. GitHub?

Git is a distributed version control system that tracks changes in source code during software development. It was created by Linus Torvalds in 2005 for development of the Linux kernel.

GitHub is a cloud-based hosting service that lets you manage Git repositories. GitHub adds many features on top of Git, including:

  • A web-based interface
  • Access control
  • Bug tracking
  • Feature requests
  • Task management
  • Wikis for project documentation
  • Pull requests for code review

Note: While Git is a command-line tool, GitHub provides a web-based graphical interface for Git operations and extends Git's functionality.

Creating Repositories

A repository (or "repo") is like a project folder that contains all of your project's files and stores each file's revision history.

Creating a new repository on GitHub

  1. Log in to your GitHub account
  2. Click the "+" icon in the top-right corner and select "New repository"
  3. Name your repository
  4. Add an optional description
  5. Choose public or private visibility
  6. Select "Add a README file" if desired
  7. Choose whether to add .gitignore and a license
  8. Click "Create repository"

Cloning an existing repository

Cloning creates a local copy of a repository on your computer.

git clone https://github.com/username/repository.git

Tip: When creating a new repository, initializing it with a README file is recommended as it provides immediate documentation and allows you to clone the repository right away.

Making Commits

A commit is a snapshot of your repository at a specific point in time. It's like saving a new version of your project.

Basic commit workflow

  1. Make changes to your files
  2. Stage the changes using git add
  3. Commit the staged changes with a message using git commit
  4. Push the commits to GitHub using git push

Git commands for committing

# Check status of your files
git status

# Stage specific file
git add filename.txt

# Stage all changes
git add .

# Commit with a message
git commit -m "Your detailed commit message here"

# Push to GitHub
git push origin main

Note: Write clear and descriptive commit messages. A good practice is to use the imperative mood (e.g., "Add feature" instead of "Added feature").

Working with Branches

Branches allow you to develop features, fix bugs, or experiment with new ideas in a contained area of your repository.

Basic branching concepts

  • Main branch: The default branch (formerly called "master")
  • Feature branches: Created to develop specific features isolated from the main branch
  • Merging: Combining changes from one branch into another

Common branch commands

# Create and checkout a new branch
git checkout -b feature-branch

# Switch to an existing branch
git checkout branch-name

# List all branches
git branch

# Push a branch to GitHub
git push origin feature-branch

# Merge a branch into your current branch
git merge branch-name

Tip: Name your branches descriptively, using slashes to categorize them (e.g., feature/user-authentication, bugfix/login-issue).

GitHub Interface Overview

Key Areas of the GitHub Interface

Feature Description
Code tab View files and folders in your repository
Issues tab Track bugs, enhancements, and other tasks
Pull Requests tab Review and discuss code changes before merging
Actions tab Set up CI/CD workflows and automation
Projects tab Project management boards
Wiki tab Documentation for your project
Insights tab Analytics and statistics for your repository
Settings tab Repository configuration options

Note: Some tabs may not be visible if certain features aren't enabled for your repository.
