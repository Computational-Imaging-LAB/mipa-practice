How to Use This Notebook

This interactive notebook contains practical exercises to help you apply what you've learned about GitHub. Each exercise builds on skills from previous ones, taking you from basic to advanced GitHub operations.

Create a Practice Repository

For these exercises, you'll need a practice repository. Create one on GitHub with the following steps:

  1. Go to GitHub and log in to your account
  2. Click the "+" icon in the top-right corner and select "New repository"
  3. Name it "mipa-practice"
  4. Add a description: "My repository for GitHub practice exercises"
  5. Choose "Public" visibility
  6. Check "Add a README file"
  7. Click "Create repository"

Basic Exercises

Exercise 1: Clone Your Repository

First, let's get a local copy of your repository.

  1. Open your terminal or command prompt
  2. Navigate to where you want to store your project
  3. Clone your repository with: git clone https://github.com/abdullahbas/mipa-practice.git
  4. Navigate into the cloned repository: cd mipa-practice

Commands Used

git clone https://github.com/abdullahbas/mipa-practice.git
cd mipa-practice

Exercise 2: Make and Commit Changes

Now let's make some changes and commit them to your repository.

  1. Create a new file called about.txt in your repository
  2. Add a few lines about yourself or the project
  3. Save the file
  4. Check the status of your repository: git status
  5. Stage the file: git add about.txt
  6. Commit your changes: git commit -m "Add about file"
  7. Push your changes to GitHub: git push

Commands Used

git status
git add about.txt
git commit -m "Add about file"
git push

Exercise 3: Create and Switch Branches

Let's practice working with branches.

  1. Create a new branch called "feature-readme-update": git checkout -b feature-readme-update
  2. Open README.md and add more information about your project
  3. Commit your changes to this branch
  4. Push the branch to GitHub: git push -u origin feature-readme-update
  5. Switch back to the main branch: git checkout main

Commands Used

git checkout -b feature-readme-update
# Edit README.md
git add README.md
git commit -m "Update README with more information"
git push -u origin feature-readme-update
git checkout main

Intermediate Exercises

Exercise 4: Create a Pull Request

Now let's create a pull request to merge your feature branch changes into the main branch.

  1. Go to your repository on GitHub
  2. Click the "Pull requests" tab
  3. Click the green "New pull request" button
  4. Set the base branch to "main" and the compare branch to "feature-readme-update"
  5. Click "Create pull request"
  6. Add a title and description for your pull request
  7. Click "Create pull request" again to submit it

Exercise 5: Review and Merge a Pull Request

Now let's review and merge the pull request you created.

  1. On the pull request page, click the "Files changed" tab to see your changes
  2. Add a comment to your own code (for practice)
  3. Go back to the "Conversation" tab
  4. Click the green "Merge pull request" button
  5. Confirm the merge by clicking "Confirm merge"
  6. Delete the branch if you don't need it anymore

Exercise 6: Resolving Merge Conflicts

Let's practice resolving merge conflicts, which occur when changes overlap.

  1. Pull the latest changes to your local repository: git pull
  2. Create another branch: git checkout -b conflict-branch
  3. Edit the first line of your README.md file and commit the changes
  4. Switch back to main: git checkout main
  5. Edit the same line of README.md differently and commit
  6. Try to merge your branch: git merge conflict-branch
  7. Resolve the conflict by editing the file (removing conflict markers)
  8. Stage the resolved file: git add README.md
  9. Complete the merge: git commit

Commands Used

git pull
git checkout -b conflict-branch
# Edit README.md
git add README.md
git commit -m "Edit README in conflict-branch"
git checkout main
# Edit same line in README.md
git add README.md
git commit -m "Edit README in main branch"
git merge conflict-branch
# Resolve conflicts in editor
git add README.md
git commit

Advanced Exercises

Exercise 7: Setting Up GitHub Actions

Let's create a simple GitHub Actions workflow.

  1. In your local repository, create a directory: mkdir -p .github/workflows
  2. Create a file named .github/workflows/hello.yml
  3. Add the following content to the file:
name: Hello World Workflow

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  greet:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v3
    - name: Run a simple greeting
      run: echo Hello, world!
  1. Commit and push this file to your repository
  2. Go to the "Actions" tab on GitHub to see your workflow run

Exercise 8: Creating GitHub Pages

Let's set up GitHub Pages to host a simple website from your repository.

  1. Create a file named index.html in your repository with some basic HTML content
  2. Commit and push the file to GitHub
  3. Go to the repository settings on GitHub
  4. Navigate to the "Pages" section in the left sidebar
  5. Under "Source", select "main" branch and "/ (root)" folder
  6. Click "Save"
  7. Wait a few minutes, then visit the published site at the URL provided

Tip: You can create a more interesting site with a template or use a static site generator like Jekyll, which GitHub Pages supports natively.

Exercise 9: Working with Issues and Projects

Let's practice using GitHub's issue tracking and project management features.

  1. Go to the "Issues" tab on your repository
  2. Create a new issue titled "Add documentation"
  3. Add a description and labels to your issue
  4. Go to the "Projects" tab and create a new project
  5. Choose the board template
  6. Add your issue to the project board
  7. Create a new branch from this issue
  8. Make changes to address the issue
  9. Create a pull request that references and closes the issue

Challenge Exercises

Challenge 1: Git Rebase

Practice using git rebase to maintain a clean commit history.

  1. Create a new branch for a feature
  2. Make several small commits
  3. Use interactive rebase to combine (squash) the commits: git rebase -i HEAD~3 (if you have 3 commits)
  4. Push your changes (you'll need to force push): git push -f

Warning: Be careful with force pushing to shared branches as it rewrites history. Only force push to your personal branches or when you're absolutely sure it's safe.

Challenge 2: GitHub API

Try using GitHub's API to interact with your repository programmatically.

  1. Generate a personal access token on GitHub (Settings → Developer settings → Personal access tokens)
  2. Use curl or another tool to fetch information about your repository:
curl -H "Authorization: token YOUR_TOKEN" \
     https://api.github.com/repos/abdullahbas/mipa-practice
  1. Try creating an issue via the API:
curl -H "Authorization: token YOUR_TOKEN" \
     -X POST \
     -d '{"title":"API Created Issue","body":"This issue was created via the GitHub API"}' \
     https://api.github.com/repos/abdullahbas/mipa-practice/issues
